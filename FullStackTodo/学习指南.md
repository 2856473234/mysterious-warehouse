# ASP.NET Core MVC 全栈项目 — 学习指南

> 目标：用最短的时间看懂你手头的 Todo 全栈项目，面试能讲清楚。
> 学习时间：约 1–2 天

---

## 第一部分：这个项目是干什么的？

一个全栈待办事项管理器：

```
你在浏览器打开页面
    → 看到待办列表
    → 输入新待办，点添加
    → 数据存到 SQLite 数据库
    → 刷新页面，数据还在
```

**全栈**的意思是：前端（你看到的页面）+ 后端（处理逻辑）+ 数据库（存数据）都是我们自己写的，而且**全部用 C#**。

## 第二部分：项目结构

```
FullStackTodo/
├── Program.cs              # 入口：启动服务器、配数据库
├── Models/
│   └── TodoItem.cs         # 数据模型（待办的"样子"）
├── Data/
│   └── AppDbContext.cs     # 数据库连接
├── Controllers/
│   └── TodoController.cs   # 控制器（处理请求的核心）
├── Views/
│   └── Todo/
│       └── Index.cshtml    # 页面模板（HTML + C# 混合）
└── wwwroot/                # 静态文件（CSS、JS）
```

### 请求是怎么跑的？（MVC 流程）

```
浏览器输入 http://localhost:5000/Todo
    → 路由找到 TodoController
    → 执行 Index() 方法
        → 从数据库查所有待办
        → 传给 Index.cshtml 视图
    → 视图生成 HTML 页面
    → 返回给浏览器显示
```

**MVC 就是 Model（数据）- View（页面）- Controller（逻辑）** 三层分开。

> 去读：`TodoController.cs` 第 14-18 行 — 最简的 Index 方法

## 第三部分：必须搞懂的三个概念

### 3.1 Controller（控制器）

控制器就是**处理请求的地方**。每个方法对应一个"操作"：

```csharp
public class TodoController : Controller
{
    // GET /Todo  → 显示列表
    public IActionResult Index()
    {
        var items = _db.TodoItems.ToList();
        return View(items);  // 把数据传给视图
    }

    // POST /Todo/Create  → 添加新待办
    public IActionResult Create(TodoItem item)
    {
        _db.TodoItems.Add(item);
        _db.SaveChanges();
        return RedirectToAction("Index");  // 回到列表页
    }
}
```

**关键理解：** 每个 public 方法就是一个"页面"或"操作"。用户在浏览器操作 → 请求发到某个方法 → 方法处理 → 返回结果。

> 去读：`TodoController.cs` 全部 — 只有 4 个方法，很简短

### 3.2 Razor 视图（.cshtml）

`.cshtml` = HTML 里可以混写 C# 代码：

```html
<!-- 纯 HTML -->
<h2>待办事项</h2>

<!-- 混入 C# 代码（@ 开头） -->
@foreach (var item in Model)
{
    <div>
        <!-- @item.Title 输出 C# 变量的值 -->
        <span>@item.Title</span>
    </div>
}

<!-- C# 条件判断 -->
@if (!Model.Any())
{
    <p>暂无数据</p>
}
```

**关键理解：** 服务器先执行 `.cshtml` 里的 C# 代码，生成纯 HTML，再发给浏览器。

> 去读：`Views/Todo/Index.cshtml` — 看看 @ 开头的地方

### 3.3 Model + EF Core（数据层）

Model 定义数据的结构，EF Core 负责存到数据库：

```csharp
// Model：定义"待办"长什么样
public class TodoItem
{
    public int Id { get; set; }           // 唯一编号
    public string Title { get; set; }     // 标题
    public bool IsDone { get; set; }      // 是否完成
    public DateTime CreatedAt { get; set; } // 创建时间
}

// EF Core 操作数据库
_db.TodoItems.Add(item);      // 插入
_db.TodoItems.ToList();       // 查询全部
_db.TodoItems.Find(id);       // 按 ID 查
_db.TodoItems.Remove(item);   // 删除
_db.SaveChanges();            // 保存修改（重要！不调这个不生效）
```

**关键理解：** EF Core 把 C# 对象自动转成 SQL 语句，你不用写 SQL。

> 去读：`Models/TodoItem.cs` — 数据模型
> 去读：`Data/AppDbContext.cs` — 数据库配置

## 第四部分：四个接口

这个项目就 4 个操作，对应 4 个 HTTP 请求：

| 操作 | HTTP 方法 | URL | Controller 方法 |
|------|-----------|-----|----------------|
| 查看列表 | GET | `/Todo` | `Index()` |
| 添加待办 | POST | `/Todo/Create` | `Create()` |
| 切换完成 | POST | `/Todo/Toggle/5` | `Toggle(5)` |
| 删除待办 | POST | `/Todo/Delete/5` | `Delete(5)` |

**全部代码不到 50 行。** 你看完就能讲清楚。

> 去读：`TodoController.cs` — 4 个方法，每个不到 10 行

## 第五部分：面试可能问的问题

### 基础题（必须能答）
1. **这个项目用了什么技术？**
   → ASP.NET Core MVC + SQLite + Razor 视图
2. **MVC 是什么？**
   → Model（数据）、View（页面）、Controller（逻辑）三层分离
3. **请求怎么走的？**
   → 浏览器发请求 → Controller 处理 → View 生成 HTML → 返回浏览器
4. **数据存在哪？**
   → SQLite 数据库，文件叫 todo.db

### 进阶题（理解了就能答）
5. **@ 符号在 .cshtml 里干什么？**
   → 切换到 C# 模式，在 HTML 里写 C# 代码
6. **为什么调了 Add 还要调 SaveChanges？**
   → Add 只是告诉 EF Core"我要加这条"，SaveChanges 才真正写入数据库
7. **RedirectToAction 是干什么？**
   → 操作完成后跳转到另一个页面（比如添加完回到列表）
