## 第一部分：项目总览（先看这个）

### 1.1 项目是干什么的？

大学选课系统的简化版。三个角色登录后干不同的事：

| 角色 | 登录方式 | 能干什么 |
|------|----------|----------|
| **学生** | 学号 + 密码 | 看可选课程 → 选课 / 看课表 / 退课 / 查成绩 |
| **教师** | 姓名 + 密码 | 看自己教什么课、看选课学生列表、录入成绩 |
| **管理员** | 用户名 + 密码 | 增删教师/课程/教室/学期、排课（给课程安排时间） |

### 1.2 项目结构

### 1.3 数据流（选课流程）

```
学生点"选课"按钮
    → StudentMainForm.DoEnroll()
    → EnrollmentService.Enroll(studentId, courseId)
        → 检查选课窗口期
        → 检查是否重复选课
        → 检查选课数量上限（5门）
        → 检查先修课是否通过
        → 检查名额是否已满
        → 检查时间是否冲突（学生）
        → 检查教师时间是否冲突
        → 全部通过 → 写入数据库
    → 返回"ok"
    → 刷新页面显示
```

## 第二部分：最核心的概念（先学这些）

### 2.1 EF Core 是什么？（最重点）

**一句话：** 让你用 C# 代码操作数据库，不用写 SQL。

#### 传统方式（写 SQL）：
```sql
INSERT INTO Students VALUES ('小明', '2024SE01', '123456', 1)
```

#### EF Core 方式（写 C#）：
```csharp
db.Students.Add(new Student { Name = "小明", ... });
db.SaveChanges();
```

#### 怎么知道数据库里有啥？
每个 `Model` 类对应一张表：
- `Student.cs` → `Students` 表
- `Course.cs` → `Courses` 表
- `Enrollment.cs` → `Enrollments` 表

#### 在代码里怎么查数据？
```csharp
using var db = new AppDbContext();  // 打开数据库连接

// 查询：找学号是 "2024SE01" 的学生
var s = db.Students.FirstOrDefault(x => x.StudentNo == "2024SE01");

// 关联查询：找到学生，同时把他的专业也查出来
var s = db.Students.Include(x => x.Major).FirstOrDefault(...);

// 条件查询：找所有软件工程专业的学生
var list = db.Students.Where(s => s.MajorId == 1).ToList();
```

> 去读：`AppDbContext.cs` — 看里面有哪几张表
> 去读：`Student.cs` — 看学生有哪些字段
> 去读：`EnrollmentService.cs` 第 12-14 行 — 看实际怎么查数据库

### 2.2 核心业务逻辑：选课的 7 道检查

全部在 `EnrollmentService.cs` 的 `Enroll()` 方法里，顺序执行：

```
1. 课程存在吗？           → if (course == null) return "课程不存在"
2. 在选课窗口期内吗？      → 当前时间要在 EnrollStart ~ EnrollEnd 之间
3. 选过了吗？             → 不能重复选同样的课
4. 超过上限了吗？         → 每学期最多选 5 门
5. 先修课过了吗？         → 如果有先修课且没及格，不能选
6. 名额满了没？           → 已选人数 >= MaxStudents
7. 时间冲突吗？           → 学生已有课的时间段不能和新课重叠
                           → 老师该时间段也不能有别的课
```

> 去读：`EnrollmentService.cs` 第 10-103 行 — 这 7 步检查完整实现

### 2.3 时间冲突检测

这个**最有可能在面试里被问到**，因为涉及到算法思维。

#### 数据怎么存的？
- `TimeSlot` 表里有 25 个时间片：周1~周5 × 每天5个时段
- 每个时间片有个 `Label`，比如 "周一1-2节"
- 一门课可以占多个时间片（比如数据课上周一1-2 + 周三3-4）

#### 冲突怎么检查？
```csharp
// 1. 先查出学生已选课程占用的所有时间片ID
var studentTimeSlotIds = 已选课程的时间片ID集合;

// 2. 再查想选的新课占用了哪些时间片
// 3. 如果有重叠 → 冲突
foreach (var ct in course.CourseTimeSlots)
{
    if (studentTimeSlotIds.Contains(ct.TimeSlotId))
    {
        return "时间冲突";
    }
}
```

**类比：** 就像你的课表——如果周一1-2节已经有一门课了，就不能再选另一门也是周一1-2节的课。

> 去读：`EnrollmentService.cs` 第 60-72 行

### 2.5 三个关键 C# 特性

#### `using var db = new AppDbContext();`
`using` 的意思是"用完自动关闭"。跟文件用完要关一样，数据库用完也要关。写了 `using`，出了大括号就自动关。

#### `List<T>` 和 `ToHashSet()`
- `List<int>` — 一个存 int 的列表
- `ToHashSet()` — 转成集合，用来查"在不在"特别快

#### LINQ 查询语法
```csharp
// 查所有姓"张"的教师
var result = db.Teachers.Where(t => t.Name.StartsWith("张")).ToList();

// 就是 C# 版的 SQL
// SELECT * FROM Teachers WHERE Name LIKE '张%'
```

5. **时间冲突怎么检测的？** → 对比两个课的 TimeSlotId 有没有交集

7. **专业课和选修课怎么区分？** → Type 字段，"专业"只有本专业能选，"选修"全校都能选
